# A validity-auditing framework for paired-state biomarkers in Parkinson's disease

Code and figure-reproduction data for the paper
*"A validity-auditing framework for paired-state biomarkers in Parkinson's disease:
subject identity, recording geometry, and baseline severity."*

The study takes a frozen paired-state retrieval pipeline (v2A) and audits it against
three independent validity boundaries — **subject identity**, **recording geometry**,
and **baseline severity** — plus a secondary levodopa–DBS alignment check.

---

## Quick start — reproduce all figures in one command

The figures regenerate from small result tables that are committed in this repo, so
**no raw data download is needed**:

```bash
pip install -r requirements.txt
python figures/make_figures.py
```

This reads everything from `figures/data/` and writes the 7 paper figures (PDF + PNG)
to `figures/output/`.

> Fonts: the script uses Times New Roman if it is installed; otherwise it falls back
> to the default serif font (Times New Roman is proprietary and not redistributed here).

```
│   ├── 1_frozen_v2A_audit/      # frozen retrieval + nulls + bootstrap + LOSO
│   ├── 2_fingerprint_boundary/  # Boundary 1: severity ladder, v2A–v2D, fingerprint gap
│   ├── 3_geometry_MAGE_G/       # Boundary 2: MAGE-G, OXF ladder, reverse transfer
│   ├── 4_symptom_axis/          # Boundary 3: symptom-specific UPDRS axes, severity confound
│   ├── 5_levodopa_DBS/          # levodopa vs 130 Hz STN-DBS alignment
│   └── 6_baselines/             # single-feature physiological baselines
├── scripts/                     # compat shim: maps the flat `from scripts.X` imports onto analysis/
├── src/                         # shared library imported by the analysis scripts
├── outputs/                     # frozen v2A artifacts + audit outputs (LOSO, v2B bootstrap)
└── analysis_plan/               # fixed symptom-axis testing procedure (Methods note)
```

Two levels of reproduction:

1. **Figures (turnkey)** — `figures/` is self-contained and runs in seconds (above).
2. **Analyses from raw data (heavy)** — the scripts in `analysis/` are the exact code
   that produced the committed result tables. They cross-import one another under the
   original flat `scripts.` namespace, which the `scripts/` shim resolves to the grouped
   files, so the whole package imports cleanly. Running them end to end additionally
   requires downloading the two public datasets (see [`DATA.md`](DATA.md)) and is provided
   for full transparency and re-execution.

## Frozen v2A artifacts

`outputs/v2A_frozen_17subjects_33pairs/` and `outputs/v2A_frozen_18subjects_34pairs/`
hold the derived CSV/JSON outputs of the frozen v2A pipeline (summary, pairs, candidate
scores, subspace and feature inventories). They are the reference the confound-aware
variants are pinned to: v2B–v2D import v2A's constants (`TOP_K`, `APERIODIC_ALPHA`) and
check them at runtime against `frozen_v2a_summary.json` (`top_k = 5`, `aperiodic_alpha =
0.5`), so a reader can verify the variants reuse v2A's fixed settings rather than retuning
them. These are derived tables only — no raw neural recordings are redistributed.

`outputs/` also holds two audit outputs that back numbers reported in the paper:
`v2A_scientific_audit_17subjects_33pairs/loso_sensitivity.csv` (the leave-one-subject-out
v2A top-1 range, [0.806, 0.903]) and `v2B_gated_scientific_audit_17subjects_33pairs/`
(the v2B–v2A subject-bootstrap deltas reported in Supplementary Table S4).

## Environment

Python 3.11+ with the packages in `requirements.txt`
(numpy, pandas, matplotlib, scipy, scikit-learn, mne, networkx, h5py, PyYAML;
pyarrow is optional). Figure reproduction needs only numpy, pandas, matplotlib.

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

## Data

Raw data are **not** included (they are large and publicly hosted). See
[`DATA.md`](DATA.md) for the two cohorts and their DOIs:
- **ds004998** — simultaneous MEG + STN-LFP (OpenNeuro).
- **OXF / Wiest et al.** — STN-LFP, off/on + 130 Hz DBS (Oxford).
